# TRACE-Sci 指标体系概览

> 基于论文 *"AI Scientist Benchmarks Should Be Prefix-Centric, Read-Only, and Provenance-Aware"* (NeurIPS 2026 Position Paper)
>
> **V1 定版**：已整合 [TRACE_SCI_METRICS_CLARIFICATIONS.md](../spec/TRACE_SCI_METRICS_CLARIFICATIONS.md) 的全部更正和补充。

---

## 一句话总结

TRACE-Sci 提出了一种**不修改被评系统、不强制统一阶段名称**的 AI Scientist 评估框架：把任意框架的原始运行转化为一张标准化的证据图，然后用 7 个固定检查点查询这张图，最终输出 4 个互补的聚合分数。

                                  │                         │
                                  │                    槽位最佳证据覆盖
                                  │                         │
                                  │               ──→ O_{C_k} 可观测性
                                  │                         │
                                  └────── × ────────────────┘
                                           │
                                    E_{C_k} 有效分数
                                           │
                      ┌────────────────────┼────────────────────┐
                      │                    │                    │
                      ▼                    ▼                    ▼
              ┌───────────────┐   ┌──────────────┐   ┌──────────────┐
              │ PGS           │   │ TFS          │   │ OBS          │
              │ 加权 E_{C0–C4}│   │ 加权 E_{C4–C6}│  │ 均匀 O_{C0–C6}│

## 四个聚合指标一览

| 指标 | 全称 | 一句话定义 | 取值范围 |
|------|------|----------|---------|
| **PGS** | Prefix Governance Score (前缀治理分数) | 研究轨迹在最终产出之前的决策质量有多高？ | [0, 1] |
| **TFS** | Terminal Fidelity Score (终端忠实度分数) | 最终论文/主张与实际证据的一致性有多强？ | [0, 1] |
| **OBS** | Observability Score (可观测性分数) | 系统在全流程中外化了多少可信的中间证据？ | [0, 1] |
| **CPS** | Counterfactual Pruning Score (反事实剪枝分数) | 如果在首个可靠红旗处停止，能节省多少 wall-clock 时间？ | [0, 1]，无红旗时 = 0 |

## V1 关键数字速查

| 项目 | 数值 |
|------|------|
| 检查点数 | 7 (C0–C6) |
| 槽位总数 | **36** |
| 叶子总数 | **41** |
| 其中 fatal 叶子 | **19** |
| 信任等级 | 4 级 (T1=1.00, T2=0.85, T3=0.60, T4=禁止) |
| 原子评分档位 | 4 档 (1.0 / 0.5 / 0.0 / 0.0+penalty) |
| RED 阈值 | E < 0.50 或 fatal leaf < 0.50 |
| Reliable Red Flag | RED 且 O ≥ 0.70 |
| CPS 成本通道 | wall-clock seconds |
| 图序列化格式 | canonical JSON |

## 六大设计原则

| 原则 | 关键词 | 含义 | 对应机制 |
|------|-------|------|---------|
| P1 | 评估前缀 | 研究轨迹在终点之前就应该可评分 | PGS |
| P2 | 只读评估 | 不修改被评框架 | 被动观察器 + TraceMap |
| P3 | 证据图归一化 | 不统一阶段名，统一证据表示 | SciTrace-IR |
| P4 | 可观测性即分数 | 缺失中间状态是可度量的弱点 | OBS + 信任分层 |
| P5 | 客观优先、必要时原子语义 | 能用脚本判的不用 LLM | 叶子树：客观叶子 + 语义叶子 |
| P6 | 审计归一化器和 Judge | 评估工具本身也要被评估 | 提取 F1、幻觉率、Judge 校准 |
